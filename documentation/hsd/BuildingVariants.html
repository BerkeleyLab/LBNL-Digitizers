Introduction

The high speed digitizer application supports several variations

  • ZCU111 or ZCU208 evaluation boards
  • AC or DC coupled ADC channels
  • Direct sampling digitizer or interleaved sampling bunch current monitor

ZCU111 or ZCU208 evaluation boards

These are completely separate Vivado/Vitis projects so the selection is made when opening the project/workspace.  The projects use symbolic links with relative paths to share common source and thus must have their top level directories located in the same directory.  The top level directories are named HighSpeedDigitizer and HSD_ZCU208, respectively.

AC or DC coupled ADC channels

In Vivado open the block design, double-click on the Zynq Ultrascale+ RF Data Convert block, then set the Link Coupling for the tiles of interest.  The LBNL analog front end card requires that all ADC tiles be set to DC coupling.  A BALUN transformer input must use AC coupling.

Direct sampling digitizer or interleaved sampling bunch current monitor

The following instructions apply to the ZCU111 variant.  The required changes for building for a ZCU208 are not shown, but should be fairly straightforward.

I've found that as of Xilinx tools 2020.1.1 (Linux) updating the hardware specification requires that the old platform must be deleted and a new platform project built from the new .xsa file.
After creating the new platform the BSP settings for the 'standalone on psu_cortexa53_0' must be modified to add the metal, lwip, xilffs, and xilpm libraries.

Building a direct sampling digitizer

  • Run the createVerilogIDX.sh script in the WorkspaceHSD/HighSpeedDigitizer/src directory to create gpioIDX.v in the firmware source directory. Change directories to the firmware source directory (HighSpeedDigitizer/HighSpeedDigitizer.srcs/sources_1/hdl). Run the startVivado.sh script to start a Vivado session.
  • Click the 'Run Synthesis' command and work through the synthesis, implementation, and bistream generation steps.
  • Export the hardware to HighSpeedDigitizer/HighSpeedDigitizer.xsa.
  • Start Vitis and set the workspace to HighSpeedDigitizer/WorkspaceHSD.
  • Update the HighSpeedDigitizerProject hardware specification from HighSpeedDigitizer/HighSpeedDigitizer.xsa.
  • Clean and build all projects.  Vitis sometimes gets dependencies wrong so it may take a couple of passes to get everything built.
  • Run 'startVivado.sh bash' in the WorkspaceHSD/HighSpeedDigitizer/src directory to start a shell session with Xilinx environment.
  • Run make in the WorkspaceHSD/HighSpeedDigitizer/scripts directory to create a bootable image.
  • Run programFlash.sh in the WorkspaceHSD/HighSpeedDigitizer/scripts directory to download the bootable image created in the previous step to the microSD card on the FPGA or mount a microSD card and copy the bootable image to it.
  • Run the pushImage.sh script in the WorkspaceHSD/HighSpeedDigitizer/scripts directory to update the bootable image on the ALS file server in the /vxboot/siochsd/head/FPGA directory.

Updating Direct Sampling Digitizer Firmware

  • Log in to an ALS machine and change directories to the /vxboot/siochsd/head/FPGA directory.
  • To update a single unit run the programFlash.sh script with the IP address of the unit to be updated as the only argument:
    sh programFlash.sh 131.243.93.161
  • The updateAll script in that directory runs the programFlash.sh script for each unit.
