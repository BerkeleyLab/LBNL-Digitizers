Introduction

This support module provides ASYN drivers to control and acquire data from the ALS bunch current monitor.  The module consists of C source, a database and engineering screens.  The monitor is based on the high speed digitizer platform and shares much of its firmware and software with the high speed digitizer application.  The following documentation describes only the records unique the the bunch current monitor.  Other records are as described in the high speed digitizer documentation

Database Development

The example application distributed with this support module comes with a database containing all the following records and EDM screens to display them.  All record names begin with the macros $(P)$(R).  Only the portion of the record name following these macros is shown in the following tables.

Using bunch current monitor support in an application

Several files need minor modifications to use bunch current monitor support in an application.

  1. Add the full path to the bunch current monitor support directory to the application configure/RELEASE file:
    BCM=xxxx/modules/instrument/bunchCurrentMonitor/<release>
    Where <release> is the release number of of the bunch current monitor support. You'll need to provide a similar line for the ASYN support module.
  2. Add instrument support to application database definition file
    The application database definition file must include the database definition files for the bunch current monitor. There are two ways that this can be done:
    • If you are building your application database definition file the old-fashioned way from an xxxInclude.dbd file you include the additional database definitions in that file:
      include "base.dbd"
      include "bunchCurrentMonitorSup.dbd"
    • If you are building your application database definition file from the application Makefile you specify the additional database definitions there:
      xxx_DBD += base.dbd
      xxx_DBD += bunchCurrentMonitorSup.dbd
  3. Add the bunchCurrentMonitorSup support libraries to the application
    You must link the bunchCurrentMonitorSup support library and the ASYN support library with the application. Add the following lines to the application Makefile:
    xxx_LIBS += bunchCurrentMonitor
    xxx_LIBS += asyn
    before the
    xxx_LIBS += $(EPICS_BASE_IOC_LIBS)
  4. Add a bunchCurrentMonitorConfigure command to the application startup script.  The arguments must appear in the order shown below.
    1. The ASYN port name.  Typically this is "BCM".
    2. The IPv4 address of the bunch current monitor FPGA.
    3. The priority at which the port threads will run.  A value of 0 results in the default medium priority.
    4. The number assigned to the first bunch.  Some sites start numbering from 0 others from 1.
    5. The number of bunches per turn.
    6. The number of samples per turn.
    7. The number of event receiver ticks per turn.
    8. The number of samples between adjacent writes to 'deinterleave' the acquired data.
    9. The event receiver code used to trigger an acquisition cycle.
    10. The event receiver code marking a true injection cycle.
    11. The number of event receiver clocks per second.
  5. Load a database file or files describing the records associated with the bunch current monitor.
